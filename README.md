# Автоматизация получения данных с API с обработкой ошибок

## О проекте

Этот проект показывает, как построить устойчивую автоматизацию в Make для работы с внешним API.

Сценарий получает данные о погоде из внешнего сервиса, обрабатывает JSON-ответ, формирует понятное сообщение для пользователя, отправляет его в Telegram и корректно обрабатывает ошибки.  
Если API временно недоступен или возвращает ошибку, сценарий не падает сразу: он делает повторные попытки, а после нескольких неудач записывает ошибку в Google Sheets и отправляет уведомление в Telegram.

---

## Цель проекта

Собрать не просто сценарий “запрос → сообщение”, а более надежную систему, которая:

- получает данные из внешнего API
- обрабатывает ответ в формате JSON
- форматирует результат в читаемый текст
- использует условия для “умного” сообщения
- повторяет запрос при ошибках
- логирует ошибки
- уведомляет о сбое

---

## Бизнес-задача

Во многих автоматизациях недостаточно просто подключить API.  
Важно, чтобы система:

- не ломалась при временных сбоях
- умела повторять запросы
- сохраняла информацию об ошибках
- уведомляла пользователя о проблеме

В этом проекте решается именно эта задача.
